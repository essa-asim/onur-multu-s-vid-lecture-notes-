# Lecture 5

## Summary
Covers the main topics discussed in this lecture.  
Gives an overview of how HDL is used in design and verification.

### HDL Design Methodology
Step-by-step approach to designing hardware using HDL.  
Includes planning, coding, testing, and refining the design.

### Verification
Ensures that the design works correctly before implementation.  
Done using simulations and testbenches.

### Verilog Syntax and Structuring
Basic rules and structure used to write Verilog code.  
Helps organize modules and connections clearly.

### Manipulating Bits
Working with bits using operations like slicing and concatenation.  
Important for controlling data at low level.

---

## Coding Styles
Different ways to write Verilog depending on abstraction level.  
Choice of style affects readability and design flexibility.

### Structural
Describes hardware by connecting smaller modules/gates together.  
Similar to building a circuit using components.

### Behavioral
Describes what the system should do, not how it is built.  
Uses high-level constructs like if-else and always blocks.

---

## Concepts
Important ideas needed for proper hardware design using HDL.  
Helps in understanding how designs are implemented and tested.

### Simulation & Synthesis
Simulation checks if the design behaves correctly before hardware implementation.  
Synthesis converts HDL code into actual hardware (gates).

### Parameterization and Timing
Parameterization allows making designs flexible and reusable.  
Timing deals with delays and clock behavior in circuits.

### Reusability
Design modules so they can be reused in different projects.  
Reduces effort and improves efficiency.
